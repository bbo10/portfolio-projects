# Data Analytics Portfolio

데이터를 통해 문제를 설명하는 것에서 끝나지 않고,  
실제 실행 가능한 전략으로 연결하는 분석가입니다.

분석 결과를 비즈니스 의사결정 기준으로 번역하고,  
실험 설계까지 포함해 “적용 가능한 수준”으로 만드는 데 집중합니다.

---

## What I Focus On

- 단순 지표 개선이 아닌 **의사결정 기준 설계**
- 분석 결과를 **실행 가능한 전략으로 연결**
- 성과를 검증할 수 있는 **실험 구조 설계**

---

## Projects

### 1. Credit Risk Policy Optimization
신용 승인 문제를 분류 모델이 아닌 손익 기반 의사결정 문제로 재정의하고,  
Expected Profit과 승인율 제약을 동시에 만족하는 정책을 설계

- Bad Approval Rate 절반 수준 감소
- Threshold 기반 승인 정책 설계 및 A/B 테스트 구조 정의

→ `project-credit/`

---

### 2. Instacart Retention Strategy
재구매율 개선을 위해 고객을 세분화하고,  
세그먼트별 행동 전략과 CRM 실행안을 설계

- 전체 고객의 54.1%를 차지하는 핵심 타겟 정의
- 구독, 번들, 추천 기반 리텐션 전략 도출

→ `project-instacart/`

---

### 3. E-commerce Growth Strategy (Jewelry)
광고 효율 하락 문제를 고객 구조, 리텐션, 상품 포트폴리오 관점에서 분석하고  
성장 전략을 재설계

- 광고 의존 성장 구조의 한계 진단
- 첫 구매 3개월 리텐션 전략 및 상품 집중 전략 제안

→ `project-jewelry/`

---

### 4. Gas Station Optimization
주유 의사결정을 가격이 아닌 실질 비용 기준으로 재정의하고  
즉시 활용 가능한 시뮬레이션 구조 설계

- 이동비 포함 Net Total Cost 기준 의사결정 모델 구축
- 기존 전략 대비 비용 절감 효과 검증

→ `project-gas_station/`

## How to Read This Repository

각 프로젝트는 다음 흐름으로 구성되어 있습니다:

1. 문제 정의 (Why)
2. 데이터 분석 (What)
3. 전략 도출 (So What)
4. 실행 및 실험 설계 (Now What)

단순 분석이 아니라,  
실제 적용 가능한 수준까지 고민한 과정을 담았습니다.
